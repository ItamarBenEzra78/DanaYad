שאלה 2

נתונה בעיית תזמון על מכונה בודדת. ניתן לקצר ג'וב אחד למחצית. האם תמיד כדאי לקצר את הג'וב הארוך ביותר?

א. עבור ∑Cj

כן, זה אופטימלי. הוכחה: בסידור SPT אופטימלי, ∑Cj = ∑(n-k+1)·t[k] כאשר t[k] הג'וב במקום ה-k. קיצור ג'וב j במחצית מקטין את tj ב-tj/2. ג'וב j תורם לפחות פעם אחת ל-∑Cj (בזמן ההשלמה שלו) וגם מעכב את כל הג'ובים שאחריו. לכן, ההפחתה הכוללת ב-∑Cj היא (n-k+1)·(tj/2) כאשר k מיקומו בסידור ה-SPT. כדי למקסם את ההפחתה, נרצה למקסם (n-k+1)·(tj/2). הג'וב הארוך ביותר הוא בעל tj הגדול ביותר. בסידור SPT הוא נמצא אחרון (מקום n), כלומר k=n ו-(n-k+1)=1. אבל אחרי הקיצור, זמן העיבוד שלו יורד ל-tj/2, וייתכן שהוא יזוז מקום בסידור. דוגמה נגדית: שני ג'ובים: t1=2, t2=10. ללא קיצור (SPT: 1,2): ∑Cj = 2 + 12 = 14. קיצור ג'וב 2 (הארוך): t2'=5. SPT: (1,2). ∑Cj = 2 + 7 = 9. חיסכון = 5. קיצור ג'וב 1: t1'=1. SPT: (1,2). ∑Cj = 1 + 11 = 12. חיסכון = 2. קיצור הארוך חסך 5 לעומת 2. נבדוק מקרה אחר: שלושה ג'ובים: t1=1, t2=1, t3=10. ללא קיצור (SPT: 1,2,3): ∑Cj = 1 + 2 + 12 = 15. קיצור ג'וב 3 (הארוך): t3'=5. SPT: (1,2,3). ∑Cj = 1 + 2 + 7 = 10. חיסכון = 5. הארוך תמיד אחרון ב-SPT (או אחרי הקיצור — עדיין מגיע אחרון או זז מקום אחד). ההפחתה = tj/2 כפול מספר הג'ובים שהוא מעכב (כולל עצמו). כן, תמיד אופטימלי לקצר את הארוך ביותר עבור ∑Cj. הוכחה פורמלית: נסמן את הג'וב הארוך ביותר k. קיצורו ב-tk/2 חוסך tk/2 מכל ג'וב שהוא מעכב. לכל ג'וב אחר j, קיצורו ב-tj/2 חוסך tj/2 מכל ג'וב שהוא מעכב. מכיוון ש-tk ≥ tj לכל j, ההפחתה בגין קיצור k היא הגדולה ביותר.

ב. עבור Lmax

לא, זה לא בהכרח אופטימלי. דוגמה נגדית: שני ג'ובים: t1=6, t2=10, d1=7, d2=20. סידור EDD: (1,2) C1=6, C2=16. L1=6-7=-1, L2=16-20=-4. Lmax = -1. קיצור ג'וב 2 (הארוך, t2'=5): סידור EDD: (1,2). C1=6, C2=11. L1=-1, L2=11-20=-9. Lmax = -1. ← לא השתנה קיצור ג'וב 1 (t1'=3): סידור EDD: (1,2). C1=3, C2=13. L1=3-7=-4, L2=13-20=-7. Lmax = -4. ← השתפר! דוגמה חזקה יותר: t1=6, t2=10, d1=5, d2=100. ללא קיצור EDD: (1,2). C1=6, C2=16. L1=1, L2=-84. Lmax=1. קיצור ג'וב 2: C1=6, C2=11. L1=1, Lmax=1. ← לא עזר קיצור ג'וב 1: C1=3, C2=13. שאלה 3

פונקצית המטרה: max(wj·Tj)

סעיף 1 — האם פונקצית המטרה מקובלת?

פונקציה מקובלת (regular) היא פונקציה לא יורדת של זמני ההשלמה Cj. כן, פונקצית המטרה מקובלת. הסבר: Tj = max(Cj - dj, 0). אם Cj גדל, Tj לא יורד (Tj עולה או נשאר 0). לכן wj·Tj הוא לא יורד ב-Cj. max של פונקציות לא יורדות הוא גם פונקציה לא יורדת. לכן max(wj·Tj) היא פונקציה מקובלת.

סעיף 2 — כיצד למצוא סידור אופטימלי?

מכיוון שהפונקציה היא min-max (ממזערים מקסימום), נשתמש באלגוריתם שמבוסס על חיפוש בינארי + בדיקת ישימות, או לחלופין: הגישה: שימוש באלגוריתם לבעיית max(wj·Tj). עבור מכונה בודדת, אפשר לסרוק את כל הסידורים (אם n קטן), או להשתמש בתכנות דינמי. באופן כללי, הבעיה היא NP-hard. אבל עבור מקרים פרטיים (כמו wj=1 לכל j, או dj=d), ניתן להשתמש ב-EDD. כשכל המשקולות שוות (wj = w): max(w·Tj) = w·Tmax, ומספיק למזער Tmax → סידור EDD.

סעיף 3 — שינוי נתונים

a. זמני היעד השתנו לערכים אחרים: הסידור עשוי להשתנות — כי הסידור תלוי בערכי dj. ערך פונקצית המטרה ישתנה — כי Tj תלוי ב-dj. b. זמני היעד קטנו כולם ב-1: הסידור לא ישתנה — כי הסדר היחסי בין ה-dj נשמר (כולם קטנו באותו ערך). ערך פונקצית המטרה ישתנה — כי Tj = max(Cj - dj, 0), ואם dj קטן ב-1, אז Lj = Cj - dj גדל ב-1. אם הג'וב כבר מאחר, הפיגור גדל ב-1. אם הג'וב לא מאחר, ייתכן שיתחיל לאחר. לכן: max(wj·Tj) יעלה (או יישאר זהה אם כל הפיגורים היו 0).

שאלה 4

סעיף 1 — הוכחה: אם tl ≤ tm וגם dl ≤ dm אז l לפני m בפתרון אופטימלי

הוכחה בעזרת החלפה זוגית (pairwise interchange): נניח סידור אופטימלי S בו ג'וב m מופיע לפני ג'וב l (בניגוד למבוקש). l ו-m לא בהכרח סמוכים. נסמן את הג'ובים ביניהם כקבוצה B. סידור S: ...m, B, l... סידור S': ...l, B, m... (החלפנו בין l ל-m) נסמן A = סכום זמני העיבוד של כל הג'ובים לפני m בסידור S. בסידור S: Cm = A + tm לכל ג'וב b ∈ B: Cb ללא שינוי (כי הם בין m ל-l) Cl = A + tm + ∑(tb עבור b∈B) + tl בסידור S': Cl' = A + tl לכל ג'וב b ∈ B: Cb' = Cb - tm + tl ≤ Cb (כי tl ≤ tm) Cm' = A + tl + ∑(tb עבור b∈B) + tm נבחן את ∑Tj: מקרה 1: ג'ובים ב-B עבור כל b ∈ B: Cb' ≤ Cb, לכן Tb' ≤ Tb. כלומר הפיגור לא גדל. מקרה 2: ג'וב l Cl' = A + tl ≤ A + tm + ∑tb + tl = Cl לכן Tl' ≤ Tl. מקרה 3: ג'וב m Cm' = A + tl + ∑tb + tm = Cl Cm = A + tm Cm' ≥ Cm (ייתכן ש-Tm גדל), אבל: Tm' = max(Cm' - dm, 0) = max(A + tl + ∑tb + tm - dm, 0) Tl = max(A + tm + ∑tb + tl - dl, 0) מכיוון ש-dl ≤ dm: Cm' - dm = A + tl + ∑tb + tm - dm Cl - dl = A + tm + ∑tb + tl - dl = Cm' - dm + (dm - dl) כלומר: Cl - dl ≥ Cm' - dm לכן: Tl ≥ Tm' סיכום: ∑Tj(S') = ∑Tj(S) - Tl - Tm + Tl' + Tm' מכיוון ש-Tl' ≤ Tl ו-Tm' ≤ Tl (ו-Tm ≥ 0): ∑Tj(S') ≤ ∑Tj(S) מכאן ש-S' לפחות טוב כמו S. לכן קיים פתרון אופטימלי בו l לפני m. ∎

סעיף 2 — כאשר dj = d לכל הג'ובים, הבעיה נפתרת בזמן פולינומיאלי

הוכחה: כאשר dj = d לכל j, מתקיים dl = dm = d לכל זוג ג'ובים l, m. אם tl ≤ tm, אז tl ≤ tm וגם dl ≤ dm (כי dl = dm = d). לפי הטענה מסעיף א', קיים פתרון אופטימלי בו l לפני m. משמעות: בפתרון אופטימלי, כל ג'וב עם זמן עיבוד קצר יותר מופיע לפני ג'וב עם זמן עיבוד ארוך יותר. זהו בדיוק סידור SPT (Shortest Processing Time). מיון n ג'ובים לפי SPT דורש O(n log n) — זמן פולינומיאלי.
לכן כאשר dj = d, הבעיה 1|dj=d|∑Tj נפתרת ע"י SPT בזמן O(n log n). ∎
